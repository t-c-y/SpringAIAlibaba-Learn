API 速查表

API / 概念一句话说明
MessageChatMemoryAdvisor把 ChatMemory 接入 ChatClient 的 Advisor
MessageWindowChatMemory滑窗记忆，只保留最近 K 轮对话
InMemoryChatMemory全量内存记忆（不推荐生产）
chat_memory_conversation_idAdvisor param，会话隔离 ID，必须从登录态取
chat_memory_response_size保留最近多少条 exchange（一问一答算一个 exchange）
JdbcChatMemory / RedisChatMemory生产用持久化实现（Spring AI 提供）

典型代码模板

this.chatClient = chatClientBuilder
    .defaultAdvisors(MessageChatMemoryAdvisor.builder(memory).build())
    .build();
// 调用时传 advisor param
spec.param("chat_memory_conversation_id", userId);

踩坑清单

  • conversationId 必须来自登录态不能让前端传任意值（否则能读别人会话）
  • 窗口大小默认 20 太大占 token；FAQ 类 bot 用 5-10 足够
  • InMemoryChatMemory 重启丢数据，生产换 JDBC/Redis
  • 敏感信息（手机号/身份证/密码）要在记忆里脱敏或不存
  • 并发写同一 conversationId 要加锁
  • 记忆满要做"会话摘要"而非粗暴清空（长对话场景）
