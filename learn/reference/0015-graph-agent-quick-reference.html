API 速查表

API / 概念一句话说明
Node一个节点做一件事；输入输出是结构化 state，不是自然语言
Edge决定从当前节点跳转到哪个下一节点
CLASSIFY 根节点唯一用 LLM 的节点，输出枚举意图，temperature=0+few-shot
END 节点结束图执行，输出 answer
maxSteps防死循环上限（推荐 ≤16）
state Map跨节点共享的上下文（intent/uid/orders/answer 等）

踩坑清单

  • 业务节点（查订单/查政策）不要调 LLM，直接查数据/规则
  • 分类节点 Prompt 要列全所有意图 enum + few-shot 示例，否则容易 FALLBACK
  • maxSteps 要限制并在超限时打错误日志
  • 状态 Map 不要无限膨胀（长对话要做摘要或清理）
  • 每个分支必须有 END 路径，不能形成环
  • 资金/退款类节点必须 HITL，不能直接执行
  • 每个节点要可单独单元测试（输入 state 断言新 state）
