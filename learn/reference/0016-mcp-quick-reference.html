踩坑清单

  • MCP Server 不能信任 Client 传的任意参数，必须做校验（和本地 @Tool 一样）
  • 远程 MCP 必须加鉴权（JWT/API Key/mTLS）
  • 工具清单过大会占满 Prompt，要按场景过滤暴露给模型
  • 跨网络调用 MCP 要设置超时、重试、熔断
  • MCP Server 版本要和 Client 兼容，schema 变更要走版本号
  • 所有 MCP 调用要打审计日志（谁、什么时间、调了什么工具、结果）
  • 教学 MockMcpServer 是 REST 模拟，真实 MCP 是 JSON-RPC，迁移时注意协议差异
