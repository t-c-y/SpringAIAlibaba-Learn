API 速查表

API / 概念一句话说明
Accept: text/event-streamSSE 响应头，curl 用 -H 设置
curl -N关闭 curl 缓冲，立即看到每一帧
.stream().content()返回 Flux，每 next 是一段 token
produces = MediaType.TEXT_EVENT_STREAM_VALUE@GetMapping 注解声明 SSE
EventSource (浏览器)new EventSource(url) 监听 message 事件
data: ... \n\nSSE 帧格式；注释行以 : 开头；[DONE] 约定结束

典型代码模板

@GetMapping(value="/chat/stream", produces=TEXT_EVENT_STREAM_VALUE)
public Flux stream(@RequestBody AskRequest req) {
  return chatClient.prompt().user(req.question()).stream().content();
}

踩坑清单

  • Nginx 反向代理默认会缓冲 SSE，必须加 proxy_buffering off
  • 移动端网络切换 SSE 不会自动恢复，要业务层做重连
  • Flux 内抛异常会直接关闭流，用 onErrorResume 降级为错误提示
  • 前端 EventSource 只支持 GET，POST 流需用 fetch + ReadableStream
  • 前端要做打字机节流（逐 token 追加太快会抖动）
