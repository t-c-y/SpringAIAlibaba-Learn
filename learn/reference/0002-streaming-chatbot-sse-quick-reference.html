流式 ChatBot 与 SSE 快速参考

用于复习阶段三：同步调用、流式调用、SSE 接口和 curl 测试方式。

同步调用

String answer = chatClient.prompt()
        .user("请解释 Spring AI Alibaba")
        .call()
        .content();

同步调用会等待模型完整生成后再返回，适合短回答和后台任务。

流式调用

Flux<String> answerStream = chatClient.prompt()
        .user("请解释 SSE 为什么适合 ChatBot")
        .stream()
        .content();

流式调用返回 Flux<String>，模型生成一部分，服务端就可以向客户端返回一部分。

SSE Controller 最小形态

@PostMapping(value = "/chat/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public Flux<String> stream(@RequestBody AskRequest request) {
    return chatClient.prompt()
            .user(request.question())
            .stream()
            .content();
}

关键点：MediaType.TEXT_EVENT_STREAM_VALUE 告诉客户端，这不是普通 JSON，而是服务端持续发送的事件流。

curl 测试

curl -N -X POST 'http://localhost:8082/chat/stream' \
  -H 'Content-Type: application/json' \
  -H 'Accept: text/event-stream' \
  -d '{"question":"请用三点解释 SSE 为什么适合 ChatBot"}'

-N 用于关闭 curl 缓冲，否则终端可能看起来不像逐步输出。

同步 vs 流式

同步

  • .call().content()
  • 一次性返回完整结果
  • 返回普通 JSON 更常见

流式

  • .stream().content()
  • 边生成边返回文本片段
  • 返回 text/event-stream

常见问题

  • 没有流式效果：检查是否使用 curl -N。
  • 接口超时：调大 DASHSCOPE_READ_TIMEOUT 并确认网络能访问 DashScope。
  • 返回不是 JSON：这是正常的，SSE 是文本事件流。
  • 本阶段没有多轮上下文：因为还没有引入 Memory。
