三种典型场景

// 1. 字符串直接渲染
String u = new PromptTemplate("请解释 {topic} 对 {aud} 的价值").render(Map.of("topic","RAG","aud","后端"));

// 2. 从 classpath 加载
@Value("classpath:prompts/lesson.st") Resource res;

// 3. 组装多消息 Prompt
Prompt p = new Prompt(List.of(sysMsg, new UserMessage(userText)));

模板治理建议

  • 一个业务场景一个模板文件，文件名与业务同名。
  • 模板里保留结构（角色/结构/边界）稳定，变量只承担"每次不同的信息"。
  • 模板上加版本号 header 注释，便于 review。
  • 灰度：v1.st / v2.st，配合流量分流。
