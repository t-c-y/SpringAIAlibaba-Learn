API 速查表

API / 概念一句话说明
SimpleVectorStore内存向量库，教学用；生产换 PGVector/Milvus
TokenTextSplitter(chunkSize, overlap, ...)按 token 切文本，默认 chunkSize=400,overlap=200
DashScopeEmbeddingModel阿里云 text-embedding-v2 等 Embedding 实现
QuestionAnswerAdvisor自动检索 topK 块塞进 Prompt
VectorStore.similaritySearch(SearchRequest)手工检索，调试召回用
SearchRequest.query(q).withTopK(k).withSimilarityThreshold(t)构建检索请求

典型代码模板

@Bean
SimpleVectorStore vectorStore(EmbeddingModel em, Resource md) {
  var store = SimpleVectorStore.builder(em).build();
  var docs = new TextReader(md).get();
  var chunks = new TokenTextSplitter(400,200,10,5000,true).split(docs);
  store.add(chunks);
  return store;
}

踩坑清单

  • SimpleVectorStore 重启丢数据，生产必须换持久化 VectorStore
  • embedding 模型维度固定，换 embedding 模型必须重建索引
  • QuestionAnswerAdvisor 有自己的 System Prompt，可能与你的 System 冲突（看 FAQ 配置）
  • 相似性阈值 threshold 默认值不一定适合你的数据，要实验
  • TokenTextSplitter 对代码/表格/标题可能切得不好，考虑用 Markdown-aware splitter
  • 多租户必须用 metadata 过滤（SearchRequest.withFilterExpression）
  • 先调用 /retrieve 接口看召回再调 ask，区分"检索差"还是"生成差"
