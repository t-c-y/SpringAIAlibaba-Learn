ChatClient 快速参考

用于复习阶段一、阶段二中最核心的模型调用链路。

最小同步调用

String answer = chatClient.prompt()
        .user("请介绍 Spring AI Alibaba")
        .call()
        .content();

prompt()

开始构造一次模型请求。可以继续追加 system、user、assistant 等消息。

user(...)

用户本次输入。对于 Web API，通常来自请求体中的问题字段。

call()

发起同步调用。线程等待模型完整生成后再继续执行。

content()

取出模型回答的文本内容，适合最小 ChatBot 和环境验证。

System Prompt 与 User Message

概念职责例子
System Prompt 定义助手身份、边界、风格和默认策略。 “你是一个企业级 Java 后端开发学习助手。”
User Message 用户本次具体问题或任务。 “请用三点说明 Spring AI Alibaba 的价值。”

企业开发提醒

  • API Key 不写入仓库，优先来自环境变量、密钥管理系统或配置中心。
  • 模型调用要封装在清晰的后端边界内，便于增加权限、审计、限流和观测。
  • 同步调用适合简单问答和后台任务；面向用户的长回答通常需要流式输出。
  • 不要把 Memory、Tool Calling、RAG 一开始全部混入，先让单轮调用稳定。

常用排查顺序

  1. 应用是否启动：检查端口和启动日志。
  2. 配置是否读取：调用 /chat/status。
  3. API Key 是否存在：确认 DASHSCOPE_API_KEY。
  4. 模型名是否正确：确认 DASHSCOPE_CHAT_MODEL 或默认值。
  5. 远程调用是否成功：调用真正的问答接口。
