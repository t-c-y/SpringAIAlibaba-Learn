API 速查表

API / 概念一句话说明
chunkSize中文技术文档 400-800；FAQ 200-400
overlapchunkSize 的 10-20%，让跨 chunk 实体不被切断
topK召回块数，3-8，与 chunkSize 反相关
similarityThresholdDashScope v2 典型 0.2-0.5；过高漏召回，过低噪声
SearchRequest.withFilterExpressionmetadata 过滤（多租户/版本/状态）先于向量检索
RerankDashScope 提供重排 API，召回 top20 再重排选 top4
MMR最大边际相关性，在相关度和多样性间平衡

典型代码模板

var req = SearchRequest.builder()
  .query(q).topK(4)
  .similarityThreshold(0.3)
  .filterExpression("tenant == '" + uid + "'").build();
var hits = vectorStore.similaritySearch(req);

踩坑清单

  • 调参一次只改一个变量，否则无法归因
  • 不要盲目加 topK 解决召回差——往往是 chunk 切得不好
  • similarityThreshold 数值因 embedding 模型而异，换模型要重新调
  • 元数据过滤永远放第一步（缩小数据规模再做向量检索）
  • 命中对了但答案错 → 是生成问题（调 Prompt/模型）；命中错了 → 检索问题
  • 评测集要分容易/歧义/超范围/对抗四层，不只测简单题
  • 生产必须做对抗测试（"忽略资料直接回答..."）
