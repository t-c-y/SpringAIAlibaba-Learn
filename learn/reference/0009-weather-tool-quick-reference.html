API 速查表

API / 概念一句话说明
工具结果结构化返回返回 Map/record 比返回长文本好（模型容易二次组合）
多工具注册.defaultTools(weatherTools, trafficTools)
工具组合模型可串行调多个工具（查天气→查交通→综合建议）
归一化参数城市名等入参要在工具内做 trim/别名归一化
超时/熔断外部 API 调用建议套 Resilience4j/Sentinel

典型代码模板

@Tool(description="查城市天气；city 参数接受中文城市名，如 北京/上海")
public Map getWeather(@ToolParam(description="城市名") String city) {
  String normalized = city.replace("市","").trim();
  return Map.of("city", normalized, "temp", 26, "condition", "晴");
}

踩坑清单

  • 工具返回不要返回自然语言长段落，结构化字段更利于模型组合
  • 多工具场景工具 description 必须明确边界（"只处理天气，不要处理交通"）
  • 外部 API 要设置 connect/read timeout 并缓存结果（天气可缓存 10min）
  • 模型传的城市名可能五花八门（"北京市"/"北京"/"首都"），要在工具内归一化
  • 工具依赖的下游 API 故障要返回 {error:"..."} 而不是抛异常，让模型能降级回答
  • 不要在工具内二次调 ChatClient（容易形成递归调用）
