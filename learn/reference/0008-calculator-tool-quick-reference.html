Tool Calling 快速参考

基本写法

@Component class Tools {
  @Tool(description = "…")
  public X doIt(@ToolParam(description = "…") Y y) { ... }
}
builder.defaultTools(new Tools()).build();

设计准则

  • 命名和描述"面向调用方"：写"什么场景下用"，不写"内部实现"。
  • 参数用具体类型（double / String / record），避免 Object。
  • 写操作要幂等 / 需要人工确认，返回值里带 status。
  • 身份类字段（userId）不放参数，走会话上下文。

错误处理

  • 业务级错误：返回结构化对象 {allowed:false, reason:…}，模型能友好化。
  • 技术级错误：抛异常，Spring AI 会把异常字符串给模型；一般会让模型再试或求助用户。
