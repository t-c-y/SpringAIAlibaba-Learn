API 速查表

API / 概念一句话说明
@Tool(description="...")标记一个方法为 LLM 可调用工具；description 是模型选工具的唯一依据
@ToolParam(description="...")给工具参数加注释，帮助模型填参
.defaultTools(MyTools.class)注册工具类；实例对象或 Bean 也可传入
.tools(tool1, tool2)单次调用动态添加工具
ToolCallingChatOptions可配置是否允许工具调用、最大工具轮次等

典型代码模板

public class CalculatorTools {
  @Tool(description="加法，返回 a+b")
  public double add(@ToolParam(description="第一个数") double a,
                   @ToolParam(description="第二个数") double b) { return a+b; }
}

踩坑清单

  • description 写得模糊是工具调用失败的第一原因（要写清什么时候用、参数含义、返回值）
  • 工具方法必须幂等/无副作用（LLM 可能重试多次）
  • 参数必须做 Java 校验（不要信任模型传值，除零、空指针、越界都要处理）
  • 工具报错要返回结构化 {error:"..."}，不要抛异常（模型不会再试）
  • defaultTools 注册的工具所有调用都可见，要做白名单/权限控制
  • 工具轮次要限次（maxToolIterations 默认 10 够用，过大容易绕圈）
  • 工具调用要打审计日志（谁在什么时间调了什么工具、参数、结果）
