API 速查表

API / 概念一句话说明
RequestGuard限流/鉴权入口，令牌桶算法（X req/min/user）
AgentRouter按意图路由到不同 Agent 实现（关键词/embedding/小分类模型）
Agent 接口String id(); String handle(uid, question);
AuditLog记录每次调用（id/at/uid/agentId/latency/q/a）
429 Too Many Requests限流拒绝返回，含 Retry-After 头
Sentinel/Nacos生产级限流熔断、配置中心

典型代码模板

@PostMapping("/platform/ask")
public Object ask(@RequestHeader("X-User-Id") String uid,
                 @RequestBody AskRequest req) {
  guard.checkAccess(uid); // 限流+鉴权
  Agent agent = router.route(req.question());
  long t0 = System.currentTimeMillis();
  String ans = agent.handle(uid, req.question());
  audit.log(uid, agent.id(), System.currentTimeMillis()-t0, req.question(), ans);
  return Map.of("success",true,"routedAgent",agent.id(),"answer",ans);
}

踩坑清单

  • 路由错误必须有 fallback Agent（兜底通用模型），不能直接报错
  • 审计日志要 PII 脱敏（手机号/身份证/邮箱打码）
  • Agent 要隔离超时（一个慢 Agent 不能卡整个平台）
  • 新 Agent 上线必须灰度（按用户/百分比放量）
  • 入口要做 Prompt 安全过滤（涉黄涉政/注入/越狱）
  • 多租户场景 Agent 清单/限流配额/审计要按 tenantId 隔离
  • metrics 维度：总请求量/错误率/各 Agent 分布/平均延迟/P99
  • 不要让 Agent 直接暴露为公共 API，必须经过平台网关
