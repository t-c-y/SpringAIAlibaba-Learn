API 速查表

API / 概念一句话说明
CallAdvisor / StreamAdvisorAdvisor 两个接口；分别拦截同步和流式
getOrder()顺序，值越小越先执行（像 Spring MVC Interceptor）
.advisors(a,b,c)单次调用添加 Advisor
.defaultAdvisors(a,b)构建时添加，所有调用生效
advisorRequest.param(k,v)在 Advisor 内读取调用方传入的参数
UserContext / ThreadLocal在 Controller 入口塞 ThreadLocal，Advisor 和 Tool 都能读

public Object advise(AdvisorRequest req, AdvisorChain chain) {
  // 请求前：往 System Prompt 追加画像

  var resp = chain.nextCall(augmented);
  // 响应后：审计日志、结果改写
  return resp;

踩坑清单

  • Advisor 顺序错乱会让 System Prompt 被覆盖，建议显式 @Order 注解
  • ThreadLocal 必须在 finally 里 clear()，否则线程池复用泄漏用户
  • Advisor 里不要做耗时 I/O（DB/外部 API），会拉长所有调用延迟
  • Advisor 抛异常会终止调用链，注意 try/catch 和错误处理
  • 同名 Advisor param 后写的会覆盖先写的
  • StreamAdvisor 和 CallAdvisor 要双写（如果 Advisor 既支持同步也支持流式）
