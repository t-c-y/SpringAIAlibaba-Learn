短期 + 长期记忆快速参考

分工原则

  • 短期：随时间失效的，例如"上一句话说了什么"。
  • 长期：稳定属性，例如姓名、岗位、偏好。
  • 写入长期一定要走确认路径（不能让模型说改就改）。

典型字段

record LearnerProfile(
  String userId,
  String name,
  String targetRole,
  List<String> mastered,
  List<String> learning,
  String weeklyBudgetHours) {}

演进路径

  1. 字段化画像（本课）。
  2. 知识条目 + 向量检索（RAG 阶段）。
  3. 结构化画像 + 事件日志（生产平台阶段）。
