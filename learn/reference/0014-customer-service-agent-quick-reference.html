踩坑清单

  • Advisor 顺序错误会导致历史被政策覆盖或工具拿不到上下文
  • 退款/赔付/开票等动作必须 HITL（先返回"确认要退款吗？"等下一轮确认）
  • 政策文档必须有版本和生效时间，过期政策要及时从向量库删除
  • 工具失败必须降级文案（"订单系统暂时开小差，稍后再试"），不要把异常抛给用户
  • 客服话术要在 System Prompt 里规范（禁止承诺具体金额/时间、不能跟用户争论）
  • 用户情绪化/投诉类问题直接转人工，模型只会火上浇油
  • 会话结束要归档或摘要存入长期记忆
