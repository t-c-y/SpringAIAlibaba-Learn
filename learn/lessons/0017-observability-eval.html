Lesson 0017 · Spring AI Alibaba

Agent 日志与评测：可观察 + 可评估

本课目标：为 Agent 装上"眼睛"和"考官"，把生产落地必需的观测/评测能力先跑起来。

配套 Demo：demos/18-observability-eval。参考卡：观测评测快速参考。

可观察

  • 每次调用记录：question / answer / latencyMs / estTokens / userId / ts。
  • 教学版存内存，生产建议：ELK / Loki + OpenTelemetry Traces / Prometheus Metrics。

LLM-as-a-judge

准备"标准问题+标准答案"，让被评模型回答，再用另一个"评审模型"判定语义等价。

数值精确 / 事实类问题不适合 LLM 评审，请用规则或人工。

动手

  1. 调 /obs/ask 若干次。
  2. 调 /obs/logs 看日志结构。
  3. 用 /obs/eval 跑一份 5 条的评测集，看通过率。

动手验证：curl 快速调用

前置条件：export DASHSCOPE_API_KEY=你的 API Key。启动对应 Demo（每个接口块第一行给出目录和端口），下面命令均可直接复制执行。

🔹 18-observability-eval（端口 8097）

启动：cd demos/18-observability-eval && mvn spring-boot:run

  1. 问答接口：自动记录 latencyMs、estTokens、回答内容到内存日志

    curl -X POST http://localhost:8097/obs/ask \
  -H 'Content-Type: application/json'\
  -d '{"userId":"u001","question":"Spring AI Alibaba 是什么"}'
# 预期：{"success":true,"answer":"...模型返回的中文回答..."}

  2. 查看最近 n 条交互日志

    curl http://localhost:8097/obs/logs?n=5
# 预期：[{id,userId,question,answer,latencyMs,at}, ...]

  3. LLM-as-Judge 批量评测：对每个用例取回答，让 judge 模型按 expected 判 pass/fail

    curl -X POST http://localhost:8097/obs/eval \
  -H 'Content-Type: application/json'\
  -d '{"cases":[{"question":"2+2等于几","expected":"4"},{"question":"中国首都在哪","expected":"北京"}]}'
# 预期：{"passCount":N,"total":N,"passRate":1.0,"details":[...]}

⚡ 3 条命令快速验证 18-observability-eval

curl -X POST http://localhost:8097/obs/ask -H 'Content-Type: application/json' -d '{"userId":"u001","question":"2+2 等于几"}'
curl "http://localhost:8097/obs/logs?n=3"
curl -X POST http://localhost:8097/obs/eval -H 'Content-Type: application/json' -d '{"cases":[{"question":"2+2等于几","expected":"4"},{"question":"中国首都在哪","expected":"北京"}]}'
