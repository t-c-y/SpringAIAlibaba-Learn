Lesson 0018 · Spring AI Alibaba

企业级 Agent 平台原型：装配前 17 课的能力

本课目标：把 Agent 抽象化，把多 Agent 路由、限流、审计装配成可扩展骨架。

配套 Demo：demos/19-agent-platform。参考卡：Agent 平台快速参考。

架构

HTTP Ingress
  ↓ RequestGuard 鉴权+限流
  ↓ AgentRouter LLM 分类
  ↓ Learning / CustomerService / Coding Agent
  ↓ 统一审计日志

关键组件

  • Agent：接口抽象，每个 Agent 有 id / description / handle。
  • AgentRouter：用轻量分类模型选 Agent。
  • RequestGuard：教学限流，生产用 Sentinel。
  • 审计：把 request/response/latency 落盘。

成本设计

路由用便宜模型，业务用贵模型。 分类只需要 1~2 个字符输出，用便宜模型省成本。

动手

  1. curl /platform/agents 看已注册 Agent。
  2. 问三类问题（代码/客服/学习），观察 routedAgent。
  3. 1 分钟内快速调 25 次，命中限流。
  4. 看 /platform/audit 是否记录完整。

动手验证：curl 快速调用

前置条件：export DASHSCOPE_API_KEY=你的 API Key。启动对应 Demo（每个接口块第一行给出目录和端口），下面命令均可直接复制执行。

🔹 19-agent-platform（端口 8098）

启动：cd demos/19-agent-platform && mvn spring-boot:run

  1. 查看已注册 Agent 列表（id/name/intents）

    curl http://localhost:8098/platform/agents
# 预期：[{"id":"smalltalk","name":"闲聊","intents":[...]}]

  2. 统一网关（需 X-User-Id 头）：自动限流（20 次/分钟/用户）+ 按意图路由 Agent + 写审计日志

    curl -X POST http://localhost:8098/platform/ask \
  -H 'Content-Type: application/json' -H 'X-User-Id: u001' \
  -d '{"question":"你好，推荐一本 Spring AI 的书"}'
# 预期：{"success":true,"routedAgent":"smalltalk","latencyMs":N,"answer":"..."}

  3. 查看最近 n 条审计记录

    curl http://localhost:8098/platform/audit?n=5
# 预期：[{id,at,userId,agentId,question,answer,latencyMs}, ...]

⚡ 3 条命令快速验证 19-agent-platform
