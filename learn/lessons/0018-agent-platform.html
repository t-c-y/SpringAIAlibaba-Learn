Lesson 0018 · Spring AI Alibaba

企业级 Agent 平台原型：装配前 17 课的能力

本课目标：把 Agent 抽象化，把多 Agent 路由、限流、审计装配成可扩展骨架。

配套 Demo：demos/19-agent-platform。参考卡：Agent 平台快速参考。

架构

HTTP Ingress
  ↓ RequestGuard 鉴权+限流
  ↓ AgentRouter LLM 分类
  ↓ Learning / CustomerService / Coding Agent
  ↓ 统一审计日志

关键组件

  • Agent：接口抽象，每个 Agent 有 id / description / handle。
  • AgentRouter：用轻量分类模型选 Agent。
  • RequestGuard：教学限流，生产用 Sentinel。
  • 审计：把 request/response/latency 落盘。

成本设计

路由用便宜模型，业务用贵模型。 分类只需要 1~2 个字符输出，用便宜模型省成本。

动手

  1. curl /platform/agents 看已注册 Agent。
  2. 问三类问题（代码/客服/学习），观察 routedAgent。
  3. 1 分钟内快速调 25 次，命中限流。
  4. 看 /platform/audit 是否记录完整。
