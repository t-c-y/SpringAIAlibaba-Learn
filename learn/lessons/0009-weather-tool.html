天气工具：外部 API 风格的 Tool

本课目标：把 Tool 从"数学函数"升级到"外部 API 风格"，学会结构化返回 + 显式 error 字段的设计。

配套 Demo：demos/10-weather-tool。参考卡：天气工具快速参考。

教学取舍

Demo 用内置 Mock 数据，不调真实气象 API。真实项目里替换成 RestClient 即可，方法签名不变。

设计要点

  • 返回 record + supported=false/reason，让模型据此说人话，而不是抛异常。
  • System Prompt 明确"必须调工具、不许口算"。
  • 参数命名与描述保持一致（city=英文小写拼音）。

动手

  1. 问"明后天去杭州合适吗？" → 模型应先调 forecast("hangzhou")。
  2. 问"成都天气如何？" → 应看到 supported=false 的话术。
  3. 把 currentWeather 加一个 timezone 字段并让模型使用。
