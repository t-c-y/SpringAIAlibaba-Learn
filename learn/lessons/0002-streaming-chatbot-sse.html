流式 ChatBot 与 SSE

本课的目标很小：把一次性返回的 ChatBot，改造成可以边生成边返回的流式接口。

配套 Demo：demos/03-streaming-chatbot。参考卡片：流式 ChatBot 与 SSE 快速参考。

你今天要获得的一个能力

能力目标：看到一个流式 ChatBot 接口时，能说清楚它如何用 ChatClient.stream() 和 HTTP SSE 把模型生成过程逐步返回给客户端。

上一课你已经掌握了同步调用：

chatClient.prompt()
        .user(question)
        .call()
        .content();

这一课只替换一个核心动作：

chatClient.prompt()
        .user(question)
        .stream()
        .content();

为什么 ChatBot 需要流式输出？

如果模型回答很长，同步接口会让用户一直等待，直到完整答案生成完毕。流式输出可以让用户更早看到内容，体验接近“助手正在打字”。

同步回答

适合短问题、后台任务、环境验证。缺点是长回答时用户一直等待。

流式回答

适合智能客服、代码解释、文档总结。模型生成一部分，用户就能看到一部分。

核心链路

HTTP POST /chat/stream
  ↓
Spring Controller
  ↓
ChatClient.prompt()
  ↓
.user(question)
  ↓
.stream()
  ↓
.content()
  ↓
Flux<String>
  ↓
text/event-stream

这里的新概念有两个：

  • Flux<String>：表示一串未来会陆续到达的字符串片段。
  • text/event-stream：HTTP SSE 的响应类型，表示服务端会持续向客户端发送事件。

SSE 是什么？

SSE 是 Server-Sent Events。它适合“服务端持续向客户端推送文本”的场景。

为什么本课用 SSE，而不是 WebSocket？

因为 ChatBot 流式回答主要是服务端向客户端单向输出。SSE 更简单，和普通 HTTP 更接近，适合作为后端流式输出的第一课。

核心代码对比

同步接口

String answer = chatClient.prompt()
        .user(request.question())
        .call()
        .content();

流式接口

@PostMapping(value = "/chat/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public Flux<String> stream(@RequestBody AskRequest request) {
    return chatClient.prompt()
            .user(request.question())
            .stream()
            .content();
}

动手练习：运行第三个 Demo

  1. 设置环境变量：export DASHSCOPE_API_KEY=你的 API Key
  2. 进入目录：cd demos/03-streaming-chatbot
  3. 启动应用：mvn spring-boot:run
  4. 检查状态：curl 'http://localhost:8082/chat/status'
  5. 测试同步接口：
    curl -X POST 'http://localhost:8082/chat/ask' -H 'Content-Type: application/json' -d '{"question":"请用三点说明为什么 ChatBot 需要流式输出"}'
  6. 测试流式接口：
    curl -N -X POST 'http://localhost:8082/chat/stream' -H 'Content-Type: application/json' -H 'Accept: text/event-stream' -d '{"question":"请用三点解释 SSE 为什么适合 ChatBot"}'

为什么仍然不引入 Memory、Tool、RAG？

因为本课只学习“传输体验”这一件事。如果同时加入多轮记忆、工具调用或知识库检索，你会很难判断问题到底出在模型调用、SSE、上下文管理，还是业务工具上。

学习边界：本课只处理单轮问题的流式返回。多轮对话会在 Memory 阶段单独学习。

  -d '{"question":"请用三点解释 SSE 为什么适合 ChatBot"}'
# 预期：text/event-stream 数据帧，逐段返回文字（data:你\ndata:好\n...\ndata:[DONE]）
