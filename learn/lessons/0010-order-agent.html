订单查询 Agent：业务 Tool + 权限守卫

本课目标：把 Tool 落到真实业务，掌握"用户身份不由模型决定"这一生产必修课。

配套 Demo：demos/11-order-agent。参考卡：订单 Agent 快速参考。

三条边界

  1. 用户身份**不允许**模型自己填，必须来自服务端会话上下文（这里用 X-User-Id + ThreadLocal）。
  2. 工具方法内部**必须做权限校验**，Prompt 是软规则不算数。
  3. 写操作要走**人工确认 + 幂等 API**，本课只覆盖只读查询。

关键代码

@Tool(description="按订单号查订单，非本人订单返回 allowed=false")
public Map<String,Object> getOrderById(String orderId) {
    String uid = UserContext.get(); // 从 ThreadLocal 拿
    ...
    if (!order.ownerUserId().equals(uid))
        return Map.of("allowed", false, "reason", "not owner");
    return Map.of("allowed", true, "order", order);
}

Controller 中 UserContext.set(userId) / finally UserContext.clear()，避免 ThreadLocal 泄漏。

动手

  1. u001 查自己订单 → 正常返回。
  2. u001 越权查 O-2001 → 应"无权限或订单不存在"。
  3. 试试不带 X-User-Id → 报错。
