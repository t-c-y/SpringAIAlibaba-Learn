Lesson 0004 · Spring AI Alibaba

结构化 JSON 输出：告别"手动 parse LLM 文本"

本课目标：会用 .entity(Class) 让模型返回可直接反序列化的 Java 对象或集合。

配套 Demo：demos/05-structured-output。参考卡片：结构化输出快速参考。

核心 API

LessonOutline o = chatClient.prompt()
    .user("请生成一份课程大纲")
    .call()
    .entity(LessonOutline.class);

List<Checkpoint> list = chatClient.prompt()
    .user("生成 5 条检查点")
    .call()
    .entity(new ParameterizedTypeReference<List<Checkpoint>>() {});

.entity() 内部做了：类型 → JSON Schema → 拼进 Prompt → BeanOutputConverter 反序列化。

Java 侧配合要点

  • record 字段名 = JSON 字段名。
  • 字段类型用具体类型（int / String / List<String>），别用 Object。
  • 温度务必调低（0.1 及以下），避免模型自由发挥破坏 JSON。

反例：不用 .entity()

Demo 还提供 /struct/raw，让你看看模型返回可能包裹 ```json、加解释、字段名不一致等麻烦。生产代码里不要这样干。

动手

  1. 启动 demos/05-structured-output
  2. 调 /struct/lesson，看返回是否是干净 JSON。
  3. 调 /struct/checkpoints，观察 List 结构。
  4. 调 /struct/raw，观察"没做转换"的原始文本。
  5. 把 record 加一个字段，重跑一遍，感受 schema 会自动带过去。
