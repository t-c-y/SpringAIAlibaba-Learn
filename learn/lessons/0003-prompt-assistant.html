Lesson 0003 · Spring AI Alibaba

Prompt 学习助手：让模型稳定回答

本课目标：会用 System Prompt 的“角色 + 输出结构 + 边界”三段式，让模型输出可预测的格式。

配套 Demo：demos/04-prompt-assistant。参考卡片：Prompt 学习助手快速参考。

核心心智模型

Prompt 工程不是让模型更聪明，而是让它更稳定。模型的能力是给定的，Prompt 用来"框住"它的输出。

三段式 System Prompt：

  1. 角色：定义 AI 扮演什么角色、服务什么听众。
  2. 输出结构：定义每段回答长什么样，是分点、卡片还是四段式。
  3. 边界：定义 AI 不该做什么（不编造、只答某话题、不确定要明说）。

核心代码

chatClient = builder.defaultSystem("""
    # 角色
    你是企业 Java + Spring AI Alibaba 学习助手。
    # 输出结构
    1) 【一句话结论】 2) 【三点要点】 3) 【最小代码示例】 4) 【常见坑】
    # 边界
    - 不确定的 API 一律说不确定。
    - 只回答与 Java/Spring/AI Agent 相关问题。
    """).build();

业务字段（topic / level / audience）应该拼进 user() 消息，不要塞进 System Prompt。

对比实验

Demo 提供两个接口：/prompt/ask（带 System Prompt） vs /prompt/ask/raw（不带）。同一个问题去问两次，一眼看出差异。

动手：跑 Demo

  1. 设置 DASHSCOPE_API_KEY
  2. cd demos/04-prompt-assistant && mvn spring-boot:run
  3. 访问 curl http://localhost:8083/prompt/status
  4. 问同一个问题两次，分别用 /prompt/ask 和 /prompt/ask/raw，对比结构
  5. 调 /prompt/ask/lesson：{"topic":"ChatMemory","level":"入门","audience":"三年 Java"}
