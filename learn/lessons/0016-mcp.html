MCP 工具集成：跨进程工具的动态发现

本课目标：理解 MCP 思想——工具通过标准协议**动态发现与调用**，而不是在编译期用 @Tool 注册。

配套 Demo：demos/17-mcp-integration。参考卡：MCP 快速参考。

MCP 是什么

  • Model Context Protocol：让工具方以标准协议对外暴露能力。
  • Agent 侧：先 list_tools 拉清单，再按 name + args 调用。
  • 好处：跨进程、跨语言，能像"应用商店"一样加载工具。

教学取舍

本课用 REST 模拟 MCP 协议：GET /mcp/tools 拉清单、POST /mcp/call 调工具。真实 MCP 是 JSON-RPC over stdio / SSE，语义完全一致。

核心流程

Agent 启动 → listTools → 拼进 System Prompt
用户提问 → LLM 输出 {"tool":..., "args":...} 或 {"final":...}
Agent 解析 → 调 MCP Server → 返回结果 → 再交给 LLM → 循环 N 次

动手

  1. 启动 Demo，curl /mcp/tools 看清单。
  2. 问 /mcp-agent/ask："帮我列出根目录文件"。
  3. 问："搜索关键字 tool 并用一句话总结"。
  4. 把 MockMcpServer 里加一个 get_time 工具，观察 Agent 自动能用。
