Lesson 0016 · Spring AI Alibaba

MCP 工具集成：跨进程工具的动态发现

本课目标：理解 MCP 思想——工具通过标准协议**动态发现与调用**，而不是在编译期用 @Tool 注册。

配套 Demo：demos/17-mcp-integration。参考卡：MCP 快速参考。

MCP 是什么

  • Model Context Protocol：让工具方以标准协议对外暴露能力。
  • Agent 侧：先 list_tools 拉清单，再按 name + args 调用。
  • 好处：跨进程、跨语言，能像"应用商店"一样加载工具。

教学取舍

本课用 REST 模拟 MCP 协议：GET /mcp/tools 拉清单、POST /mcp/call 调工具。真实 MCP 是 JSON-RPC over stdio / SSE，语义完全一致。

核心流程

Agent 启动 → listTools → 拼进 System Prompt
用户提问 → LLM 输出 {"tool":..., "args":...} 或 {"final":...}
Agent 解析 → 调 MCP Server → 返回结果 → 再交给 LLM → 循环 N 次

动手

  1. 启动 Demo，curl /mcp/tools 看清单。
  2. 问 /mcp-agent/ask："帮我列出根目录文件"。
  3. 问："搜索关键字 tool 并用一句话总结"。
  4. 把 MockMcpServer 里加一个 get_time 工具，观察 Agent 自动能用。

动手验证：curl 快速调用

前置条件：export DASHSCOPE_API_KEY=你的 API Key。启动对应 Demo（每个接口块第一行给出目录和端口），下面命令均可直接复制执行。

🔹 17-mcp-integration（端口 8096）

启动：cd demos/17-mcp-integration && mvn spring-boot:run

  1. 【MCP 服务端】查看可用工具清单（list_files / search_docs）

    curl http://localhost:8096/mcp/tools
# 预期：[{"name":"list_files",...},{"name":"search_docs",...}]

  2. 【MCP 服务端】直接调用指定工具（不经过模型，用于观察协议）

    curl -X POST http://localhost:8096/mcp/call \
  -H 'Content-Type: application/json'\
  -d '{"toolName":"search_docs","arguments":{"q":"advisor"}}'
# 预期：{"ok":true,"q":"advisor","hits":[{"title":"advisor",...}]}

  3. 【MCP 客户端 Agent】模型自主决定何时调用哪个 MCP 工具，多轮调用直至输出 final

    curl -X POST http://localhost:8096/mcp-agent/ask \
  -H 'Content-Type: application/json'\
  -d '{"question":"帮我在知识库里搜索 ChatClient 是什么，然后列出根目录文件"}'
# 预期：{"success":true,"steps":N,"answer":"..."}

⚡ 3 条命令快速验证 17-mcp-integration

curl http://localhost:8096/mcp/tools
curl -X POST http://localhost:8096/mcp/call -H 'Content-Type: application/json' -d '{"toolName":"search_docs","arguments":{"q":"advisor"}}'
curl -X POST http://localhost:8096/mcp-agent/ask -H 'Content-Type: application/json' -d '{"question":"帮我搜索知识库里 ChatClient 的说明"}'
