学习顾问 Memory：短期会话 + 长期画像

本课目标：在多轮记忆基础上叠加"用户画像"这一层长期记忆，理解两级记忆的分工。

配套 Demo：demos/08-learning-advisor。参考卡：学习顾问快速参考。

两层记忆

短期 / 会话

  • ChatMemory 存储
  • conversationId 隔离
  • "这次会话说过的话"

长期 / 画像

  • LearnerProfileStore 存储
  • userId 隔离
  • "稳定的用户属性"

怎么把画像给模型

不是每轮都塞 user 消息，而是**渲染进 System Prompt**：

String system = """
你是学习顾问，服务对象画像：

%s

回答规则：...
""".formatted(profile.toSystemFragment());

动手

  1. 建画像：POST /advisor/profile
  2. 问"下周该学什么？"，观察建议是否基于画像。
  3. 不建画像直接问，对比差异。
  4. 修改画像的 targetRole，再问同一问题。

动手验证：curl 快速调用

前置条件：export DASHSCOPE_API_KEY=你的 API Key。启动对应 Demo（每个接口块第一行给出目录和端口），下面命令均可直接复制执行。

🔹 08-learning-advisor（端口 8087）

启动：cd demos/08-learning-advisor && mvn spring-boot:run

  1. 创建或更新学员画像

    curl -X POST http://localhost:8087/advisor/profile \
  -H 'Content-Type: application/json'\
  -d '{"userId":"u001","name":"小明","targetRole":"AI 应用工程师","mastered":["Spring Boot","MyBatis"],"learning":["RAG","Tool Calling"],"weeklyBudgetHours":8}'
# 预期：{"success":true,"answer":"...模型返回的中文回答..."}

  2. 查看学员画像

    curl http://localhost:8087/advisor/profile?userId=u001
# 预期：{"userId":"u001","name":"小明",...}

  3. 删除学员画像

    curl -X DELETE http://localhost:8087/advisor/profile?userId=u001
# 预期：{"success":true}

  4. 带画像的个性化答疑（conversationId 可选，默认=userId）

    curl -X POST http://localhost:8087/advisor/ask \
  -H 'Content-Type: application/json'\
  -d '{"userId":"u001","question":"根据我的情况，下一步应该学什么"}'
# 预期：{"success":true,"answer":"...模型返回的中文回答..."}

⚡ 3 条命令快速验证 08-learning-advisor

curl -X POST http://localhost:8087/advisor/profile -H 'Content-Type: application/json' -d '{"userId":"u001","name":"小明","targetRole":"AI 工程师","mastered":["Spring Boot"],"learning":["RAG"],"weeklyBudgetHours":8}'
curl "http://localhost:8087/advisor/profile?userId=u001"
