RAG 参数调优：把感觉变成实验

本课目标：会为不同的 (chunk, overlap) 组合建 profile，对比不同 topK / threshold 的召回和生成质量。

配套 Demo：demos/14-rag-tuning。参考卡：RAG 调参快速参考。

四大调参维度

  • chunkSize：太大丢粒度，太小语义碎裂。
  • overlap：≈ chunk 的 20~30%，兜住"关键句被切断"。
  • topK：太小漏，太大噪声挤掉有用片段。
  • similarityThreshold：过滤低质量命中的兜底。

评估方法

准备 10~20 条"标准问题 + 标准答案"，逐 profile 跑一遍，记 hit@k、准确率、成本、延迟。

| profile | 命中数 | 相关度 | 回答质量 | Prompt token |
| 200-50  |        |        |          |               |
| 400-100 |        |        |          |               |
| 800-200 |        |        |          |               |

动手

  1. POST /rag/rebuild?chunk=200&overlap=50
  2. POST /rag/rebuild?chunk=800&overlap=200
  3. 用同一问题跑 /rag/retrieve 两次，比命中差异。
  4. 用 /rag/ask 比最终回答差异。

动手验证：curl 快速调用

前置条件：export DASHSCOPE_API_KEY=你的 API Key。启动对应 Demo（每个接口块第一行给出目录和端口），下面命令均可直接复制执行。

🔹 14-rag-tuning（端口 8093）

启动：cd demos/14-rag-tuning && mvn spring-boot:run

  1. 按指定 chunk/overlap 重新构建一个向量库 profile（启动时默认有 400-200）

    curl -X POST http://localhost:8093/rag/rebuild?chunk=400&overlap=200
# 预期：{"profile":"400-200","chunkSize":400,"overlap":200,"totalChunks":N}

  2. 查看已构建的所有 chunk-overlap 配置

    curl http://localhost:8093/rag/profiles
# 预期：{"profiles":["400-200",...]}

  3. 对指定 profile 做检索（可配 topK/threshold，用于对比召回效果）

    curl -X POST http://localhost:8093/rag/retrieve \
  -H 'Content-Type: application/json'\
  -d '{"profile":"400-200","question":"RAG 怎么调优","topK":4,"similarityThreshold":0.0}'
# 预期：{"hits":[{"source":"...","score":0.xx,"text":"..."}...],"question":"..."}

  4. 在指定 profile 上做 RAG 问答（代码手工拼装 Prompt 展示原理）

    curl -X POST http://localhost:8093/rag/ask \
  -H 'Content-Type: application/json'\
  -d '{"profile":"400-200","question":"chunk size 怎么选？"}'
# 预期：{"success":true,"answer":"...模型返回的中文回答..."}

⚡ 3 条命令快速验证 14-rag-tuning

curl -X POST "http://localhost:8093/rag/rebuild?chunk=400&overlap=200"
curl http://localhost:8093/rag/profiles
curl -X POST http://localhost:8093/rag/ask -H 'Content-Type: application/json' -d '{"profile":"400-200","question":"怎样选 chunk size"}'
