智能客服 Agent：Memory + RAG + Tool 三合一

本课目标：把前面学的三大能力合到一个 Agent 上，理解 Advisor 顺序和"读工具/写工具"的分层。

配套 Demo：demos/15-customer-service-agent。参考卡：综合 Agent 快速参考。

数据流

用户消息
  ↓ MessageChatMemoryAdvisor 拼历史
  ↓ QuestionAnswerAdvisor 检索知识库
  ↓ LLM 决策
  ↓ Tool (查订单 / 提交退款)
  ↓ 生成回答
  ↓ 记忆写回

关键设计

  • 知识库放"政策类"内容（退款/物流/保修）。
  • 业务动作走 Tool，Tool 内做权限校验。
  • 写操作（退款）返回 pending，明确告知"等人工审核"。

动手

  1. u001 问"我有哪些订单？"→ Tool 命中。
  2. u001 问"O-1002 已签收 3 天能退吗？"→ RAG + Tool 都命中。
  3. u001 说"帮我退 O-1002"→ Tool 返回 pending。
  4. u001 越权查 O-2001 → 返回无权限。

动手验证：curl 快速调用

前置条件：export DASHSCOPE_API_KEY=你的 API Key。启动对应 Demo（每个接口块第一行给出目录和端口），下面命令均可直接复制执行。

🔹 15-customer-service-agent（端口 8094）

启动：cd demos/15-customer-service-agent && mvn spring-boot:run

  1. 客服 Agent（需 X-User-Id 头）：自动选择走政策 RAG / 订单工具 / 普通闲聊，并带会话记忆

    curl -X POST http://localhost:8094/cs/ask \
  -H 'Content-Type: application/json' -H 'X-User-Id: u001' \
  -d '{"conversationId":"c1","question":"我想退订单 SO-1001，退款多久到账"}'
# 预期：{"success":true,"answer":"...模型返回的中文回答..."}
