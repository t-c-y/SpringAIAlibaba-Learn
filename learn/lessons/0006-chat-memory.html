多轮对话 Memory：会话上下文的第一课

本课目标：会用 ChatMemory + MessageChatMemoryAdvisor 给 ChatBot 装上记忆，通过 conversationId 隔离用户。

配套 Demo：demos/07-chat-memory。参考卡：Memory 快速参考。

三个组件

ChatMemory memory = new InMemoryChatMemory();
ChatClient client = builder.defaultAdvisors(
    MessageChatMemoryAdvisor.builder(memory).build()
).build();

client.prompt().user(q).advisors(a -> a
    .param(AbstractChatMemoryAdvisor.CHAT_MEMORY_CONVERSATION_ID_KEY, cid)
    .param(AbstractChatMemoryAdvisor.CHAT_MEMORY_RETRIEVE_SIZE_KEY, 20))
.call().content();

Advisor 干了什么

  1. 请求前：按 conversationId 从 ChatMemory 读最近 N 条历史消息，拼进 Prompt。
  2. 请求后：把最新一轮 user / assistant 写回 ChatMemory。

会话隔离

不传 conversationId 就默认合到 default 会话，用户之间会串数据。 生产必须由服务端根据用户 + 会话槽生成。

动手

  1. 会话 A 说"我叫小张"，再问"我叫什么？" → 应答对。
  2. 会话 B 问"我叫什么？" → 应回答不知道。
  3. /chat/history?conversationId=userA 看历史。
  4. /chat/reset?conversationId=userA 清空后再问。
