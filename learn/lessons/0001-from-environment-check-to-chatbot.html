Lesson 0001 · Spring AI Alibaba

从环境检查到最小 ChatBot

本课的目标很小：把你已经拥有的 Spring Boot 后端经验，连接到 Spring AI Alibaba 的第一个业务接口。

配套 Demo：demos/01-environment-check、demos/02-chatbot。参考卡片：ChatClient 快速参考。

你今天要获得的一个能力

能力目标：看到一段 Spring AI Alibaba 的 ChatClient 代码时，能说清楚它如何把 HTTP 请求转成一次大模型调用。

你已经熟悉 Java / Spring Boot，所以我们不从 Controller、Maven、REST API 重新讲起。你只需要把下面这条新链路装进已有后端知识里：

HTTP 请求
  ↓
Spring Controller
  ↓

  ↓

  ↓

为什么先做环境检查，再做 ChatBot？

AI 应用的第一类问题通常不是“业务逻辑写错”，而是调用链路没有打通：API Key 没配、模型名写错、账号没权限、网络不通、依赖版本不匹配。

阶段一：环境检查

/env/status 不调用模型，只检查本地配置；/env/ping 真正调用一次模型。

阶段二：最小 ChatBot

/chat/ask 开始接收用户问题，并返回模型回答。它才是业务接口的雏形。

企业开发里，这个顺序很重要：先证明基础设施可用，再叠加业务能力。

核心代码只看四步

String answer = chatClient.prompt()
        .user(request.question())
        .call()
        .content();
  1. prompt()：开始构造一次模型请求。
  2. user(...)：放入用户本次问题。
  3. call()：执行同步模型调用。
  4. content()：取出模型回答文本。

注意：这里还没有 Memory，所以它不是多轮对话；还没有 Tool Calling，所以模型不能真正操作外部系统；还没有 RAG，所以模型不会自动查你的企业知识库。

System Prompt 是默认策略

在 demos/02-chatbot 中，构造方法里设置了 defaultSystem(...)。它不是用户问题，而是这个 ChatBot 的默认角色和行为边界。

this.chatClient = chatClientBuilder
        .defaultSystem("你是一个企业级 Java 后端开发学习助手...")
        .build();

你可以把它类比为后端服务的“默认策略配置”：用户每次问什么会变，但系统希望助手遵守的身份、风格、边界相对稳定。

动手练习：运行第二个 Demo

  1. 设置环境变量：export DASHSCOPE_API_KEY=你的 API Key
  2. 进入目录：cd demos/02-chatbot
  3. 启动应用：mvn spring-boot:run
  4. 检查状态：curl 'http://localhost:8081/chat/status'
  5. 发起问答：
    curl -X POST 'http://localhost:8081/chat/ask' -H 'Content-Type: application/json' -d '{"question":"请用三点说明 Spring AI Alibaba 在企业开发中的价值"}'
