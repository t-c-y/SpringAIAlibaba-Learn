PDF 知识库：按页引用与上传灌库

本课目标：会用 PagePdfDocumentReader 处理 PDF，让回答能标注 [page N] 引用。

配套 Demo：demos/13-pdf-rag。参考卡：PDF RAG 快速参考。

关键差异

  • 相比 TextReader，PagePdfDocumentReader 会在 Document metadata 上写入 page_number。
  • System Prompt 里可要求模型在回答末尾用 [page N] 引用来源，实现可追溯。
  • 扫描版 PDF 需要先 OCR，本课不覆盖。

核心代码

PagePdfDocumentReader reader = new PagePdfDocumentReader(pdf,
    PdfDocumentReaderConfig.builder()
        .withPagesPerDocument(1)
        .build());
List<Document> docs = reader.get();
docs.forEach(d -> d.getMetadata().put("file_name", fileName));
vectorStore.add(new TokenTextSplitter(400, 200, ...).apply(docs));

动手

  1. 上传一份产品手册 PDF：curl -F 'file=@x.pdf' /pdf/upload
  2. 问一个只有 PDF 里才有的问题，观察回答有 [page N]。
  3. 调 /pdf/retrieve，看命中的 page 是否与回答一致。

动手验证：curl 快速调用

前置条件：export DASHSCOPE_API_KEY=你的 API Key。启动对应 Demo（每个接口块第一行给出目录和端口），下面命令均可直接复制执行。

🔹 13-pdf-rag（端口 8092）

启动：cd demos/13-pdf-rag && mvn spring-boot:run

  1. 上传 PDF 并向量化（multipart/form-data，字段名 file）

    curl -X POST http://localhost:8092/pdf/upload \
  -F 'file=@/absolute/path/to/your.pdf'
# 示例：curl -X POST http://localhost:8092/pdf/upload -F 'file=@$HOME/Downloads/sample.pdf'
# 预期：{"success":true,"fileName":"xxx.pdf","chunks":N}

  2. 按问题检索相关页（page/text/source）

    curl -X POST http://localhost:8092/pdf/retrieve \
  -H 'Content-Type: application/json'\
  -d '{"question":"这篇文档讲了什么"}'
# 预期：{"hits":[{"source":"...","score":0.xx,"text":"..."}...],"question":"..."}

  3. PDF RAG 问答，System Prompt 要求模型输出 [page N] 引用

    curl -X POST http://localhost:8092/pdf/ask \
  -H 'Content-Type: application/json'\
  -d '{"question":"文档第 3 页讲了什么？请标注页码"}'
# 预期：{"success":true,"answer":"...模型返回的中文回答..."}

⚡ 3 条命令快速验证 13-pdf-rag

# 先准备一个 PDF，然后上传
curl -X POST http://localhost:8092/pdf/upload -F 'file=@/absolute/path/to/your.pdf'
curl -X POST http://localhost:8092/pdf/ask -H 'Content-Type: application/json' -d '{"question":"这篇文档讲了什么"}'
