# Spring AI Alibaba AI Agent 学习路线规划

> 适用对象：已经掌握 Java、Spring Boot、Maven、REST API 等 Java 技术栈，但不了解 Spring AI Alibaba，希望系统学习 AI Agent 开发，并逐步具备企业级 Agent 架构设计能力的开发者。
>
> 规划目标：只做学习方案、阶段步骤、Demo 项目规划和相关技术说明，不包含代码实现。

## 学习前提

本路线默认你已经具备以下能力：

- Java 17+ 基础和常用 API。
- Spring Boot Web 开发经验。
- Maven / Gradle 基础。
- REST API、JSON、HTTP 基础。
- 常见工程分层：Controller、Service、Repository、Configuration。
- 基本数据库、缓存、日志、配置文件经验。

因此，本路线不再安排 Java 和 Spring Boot 入门内容，重点直接放在：

- Spring AI Alibaba 基础能力。
- 大模型应用开发方法。
- ChatBot 到 AI Agent 的演进。
- Tool Calling、RAG、Graph、MCP。
- 企业级 AI Agent 工程化和架构设计。

# 阶段 1：Spring AI Alibaba 基础认知与环境准备

## 建议时间

3～5 天

## 阶段目标

快速理解 Spring AI Alibaba 是什么、解决什么问题、和 Spring AI / DashScope / 百炼的关系，并准备本地开发环境。

## 学习步骤

1. 了解 Spring AI Alibaba 的定位：基于 Spring AI，面向阿里云百炼 / DashScope 的 AI 应用开发框架。
2. 理解它能做什么：ChatBot、Embedding、RAG、Tool Calling、Graph、MCP、多模型应用。
3. 了解 Spring AI Alibaba 与 Spring Boot 自动配置的关系。
4. 申请 DashScope / 百炼 API Key。
5. 配置本地环境变量。
6. 阅读官方 ChatBot 入门文档。
7. 浏览官方 examples 仓库，知道有哪些可参考 Demo。

## Demo 项目规划

### Demo 1-1：Spring AI Alibaba 环境检查 Demo

目的：验证本地开发环境、API Key 和模型配置可用。

功能规划：

- 创建最小 Spring Boot 应用。
- 引入 Spring AI Alibaba DashScope Starter。
- 配置 API Key 和默认模型。
- 启动应用，确认自动配置正常加载。

需要掌握的技术：

- Spring AI Alibaba Starter
- DashScope / 百炼 API Key
- Spring Boot 自动配置
- application.yml 模型配置
- 环境变量管理

## 阶段完成标准

- 能说明 Spring AI Alibaba 的作用。
- 能说明 Spring AI Alibaba 和 Spring AI 的关系。
- 能准备 API Key 和模型配置。
- 能启动一个最小 AI 应用工程。

# 阶段 2：ChatClient 与 ChatBot 入门

## 建议时间

1 周

## 阶段目标

掌握 Spring AI Alibaba 中最基础的大模型调用方式，跑通同步问答和流式问答。

## 学习步骤

1. 学习 `ChatClient` 的作用。
2. 学习如何构造 Prompt。
3. 学习同步调用流程。
4. 学习流式调用流程。
5. 学习 System Prompt 的作用。
6. 学习模型参数配置，例如模型名、temperature 等。
7. 学习接口层如何封装 AI 能力。

## Demo 项目规划

### Demo 2-1：最小 ChatBot Demo

目的：掌握 Spring AI Alibaba 调用大模型的最小闭环。

功能规划：

- `/chat`：普通问答接口。
- 支持通过请求参数传入用户问题。
- 设置默认系统角色。
- 从环境变量读取 API Key。
- 从配置文件读取模型名。

需要掌握的技术：

- `ChatClient`
- 同步模型调用
- System Prompt
- 模型配置
- REST API 与大模型调用结合

### Demo 2-2：流式 ChatBot Demo

目的：掌握流式输出，理解大模型应用中的用户体验优化。

功能规划：

- `/chat/stream`：SSE 流式问答接口。
- 用户提问后逐段返回模型内容。
- 对比普通接口和流式接口的体验差异。

需要掌握的技术：

- `ChatClient.stream()`
- SSE
- Flux 流式返回
- curl 流式测试
- 前后端流式交互基础

## 阶段完成标准

- 能独立开发一个 Spring AI Alibaba ChatBot。
- 能解释同步输出和流式输出的区别。
- 能通过配置切换模型。
- 能使用 System Prompt 控制模型角色。

# 阶段 3：Prompt 工程与结构化输出

## 建议时间

2 周

## 阶段目标

掌握如何稳定控制模型回答的角色、格式、风格和输出结构，为后续 Agent 决策和业务集成打基础。

## 学习步骤

1. 学习 Prompt 的基本组成：角色、任务、上下文、约束、输出格式。
2. 学习 System Prompt、User Prompt、Assistant Message 的职责划分。
3. 学习 Prompt 模板化。
4. 学习 Few-shot 示例提示。
5. 学习要求模型输出 JSON。
6. 学习结构化输出结果校验。
7. 学习 Prompt 版本管理的基本思想。
8. 学习 Prompt 与代码、配置、数据库的存放边界。

## Demo 项目规划

### Demo 3-1：学习助手 Prompt Demo

目的：掌握通过 Prompt 控制模型角色和回答风格。

功能规划：

- 将模型固定为 Spring AI Alibaba 学习助手。
- 要求回答包含“结论、步骤、注意事项”。
- 支持不同用户阶段：小白、进阶、高级。
- 输出风格要求简洁、可执行。

需要掌握的技术：

- System Prompt 设计
- User Prompt 设计
- Prompt 约束
- 回答风格控制
- Prompt 调试

### Demo 3-2：结构化 JSON 输出 Demo

目的：掌握让模型输出可被程序解析的数据结构。

功能规划：

- 输入一个学习主题。
- 输出 JSON 格式学习计划。
- JSON 包含阶段、目标、任务、预计时间。
- 后端对 JSON 字段做校验。
- 解析失败时进行兜底处理。

需要掌握的技术：

- JSON 输出约束
- 结构化结果解析
- 字段校验
- 输出容错
- 业务 DTO 映射

### Demo 3-3：Prompt 模板管理 Demo

目的：理解 Prompt 不能散落在代码中，需要模板化和版本化管理。

功能规划：

- 将 Prompt 放到配置文件或模板文件中。
- 支持变量替换。
- 给 Prompt 增加版本号。
- 记录不同版本 Prompt 的效果差异。

需要掌握的技术：

- Prompt 模板
- 配置化管理
- 变量替换
- Prompt 版本意识
- Prompt 可维护性设计

## 阶段完成标准

- 能写出结构清晰的 Prompt。
- 能让模型稳定输出指定格式。
- 能理解为什么企业项目需要 Prompt 管理。
- 能对模型输出做基本校验和兜底。

# 阶段 4：会话记忆 Memory

## 建议时间

1～2 周

## 阶段目标

让 ChatBot 支持多轮对话，理解上下文管理、会话隔离和记忆成本。

## 学习步骤

1. 学习 `ChatMemory` 的作用。
2. 学习会话 ID 的设计。
3. 学习短期上下文和长期记忆的区别。
4. 学习多用户会话隔离。
5. 学习上下文窗口限制。
6. 学习历史消息清理策略。
7. 学习历史摘要和关键信息提取。
8. 学习敏感信息不要长期进入上下文的原则。

## Demo 项目规划

### Demo 4-1：多轮对话 Demo

目的：掌握基础会话记忆能力。

功能规划：

- 请求中携带 `chatId`。
- 同一个 `chatId` 可以连续追问。
- 不同 `chatId` 的历史相互隔离。
- 支持清空指定会话历史。

需要掌握的技术：

- ChatMemory
- Conversation ID
- 多轮上下文
- 会话隔离
- 历史消息管理

### Demo 4-2：学习顾问记忆 Demo

目的：理解记忆在真实 Agent 场景中的价值。

功能规划：

- 用户第一次输入自己的基础、目标和学习时间。
- 后续提问时，助手能基于这些信息给建议。
- 支持更新学习目标。
- 支持总结用户当前学习状态。

需要掌握的技术：

- 用户画像
- 上下文注入
- 记忆更新
- 历史信息摘要
- 个性化回答

## 阶段完成标准

- 能开发支持多轮对话的 ChatBot。
- 能区分不同用户和不同会话。
- 能说明上下文过长会带来的成本和效果问题。
- 能设计简单的会话清理和摘要策略。

# 阶段 5：Tool Calling 工具调用

## 建议时间

3～4 周

## 阶段目标

让模型不只会聊天，还能调用 Java 方法或业务服务完成实际任务，正式进入 AI Agent 开发阶段。

## 学习步骤

1. 学习 Tool Calling 的基本原理。
2. 学习如何把 Java 方法暴露给模型调用。
3. 学习工具名称、工具描述、参数描述的重要性。
4. 学习工具参数校验和返回值设计。
5. 学习工具调用失败时的兜底策略。
6. 学习工具方法的权限校验。
7. 学习工具调用审计日志。
8. 学习高风险操作必须人工确认的原则。
9. 学习模型判断是否调用工具的行为特点。

## Demo 项目规划

### Demo 5-1：计算器工具 Demo

目的：用低风险场景理解 Tool Calling 的基本流程。

功能规划：

- 提供加减乘除工具。
- 用户用自然语言提问。
- 模型判断是否需要调用计算工具。
- 返回计算结果和解释。

需要掌握的技术：

- Tool Calling
- 工具参数定义
- 工具返回值
- 模型自动选择工具
- 工具调用调试

### Demo 5-2：天气查询工具 Demo

目的：理解模型调用外部 API 的场景。

功能规划：

- 用户询问某城市天气。
- 模型提取城市参数。
- 调用天气查询工具。
- 返回天气摘要。
- API 失败时返回友好提示。

需要掌握的技术：

- 外部 HTTP API 调用
- 参数提取
- 工具异常处理
- 结果摘要生成
- 外部依赖兜底

### Demo 5-3：订单查询 Agent Demo

目的：理解 Tool Calling 在业务系统中的应用方式。

功能规划：

- 用户查询订单状态。
- 模型调用订单查询工具。
- 后端模拟订单服务。
- 查询前校验用户身份。
- 记录工具调用日志。
- 对订单号、用户 ID 等参数做校验。

需要掌握的技术：

- 业务服务封装
- 权限校验
- 参数校验
- 审计日志
- 工具调用安全边界

## 阶段完成标准

- 能开发会调用工具的 Agent。
- 能设计工具参数和返回结构。
- 能理解 Tool Calling 的安全风险。
- 能说明哪些操作必须二次确认。
- 能将普通业务服务封装成 Agent 可调用工具。

# 阶段 6：RAG 知识库问答

## 建议时间

4～5 周

## 阶段目标

让 Agent 可以基于企业私有文档回答问题，降低幻觉，提高答案可追溯性。

## 学习步骤

1. 学习 RAG 基本流程。
2. 学习文档加载。
3. 学习文档切分。
4. 学习 Embedding。
5. 学习向量数据库或向量存储。
6. 学习相似度检索。
7. 学习 topK、chunk size、相似度阈值。
8. 学习将检索结果拼接到 Prompt。
9. 学习答案引用来源。
10. 学习知识库权限控制。
11. 学习 RAG 效果评估和调优。

## Demo 项目规划

### Demo 6-1：Markdown 知识库助手 Demo

目的：掌握最小 RAG 闭环。

功能规划：

- 导入 Markdown 文档。
- 对文档进行切分。
- 将切分内容向量化。
- 用户提问时检索相关片段。
- 模型基于片段回答。
- 回答中附带来源。

需要掌握的技术：

- Document Loader
- Text Splitter
- Embedding
- Vector Store
- Retriever
- RAG Prompt
- 引用来源

### Demo 6-2：PDF 知识库助手 Demo

目的：掌握更接近企业真实资料的文档问答。

功能规划：

- 导入 PDF 文档。
- 解析 PDF 文本。
- 切分并入库。
- 支持基于 PDF 内容问答。
- 返回页码或文档片段来源。

需要掌握的技术：

- PDF 文档解析
- 文档元数据
- 来源追踪
- 文档清洗
- 非结构化数据处理

### Demo 6-3：知识库参数调优 Demo

目的：理解 RAG 效果不是接入向量库就结束，需要持续调优。

功能规划：

- 对比不同 chunk size。
- 对比不同 topK。
- 对比不同相似度阈值。
- 记录回答准确性。
- 分析检索不到、检索不准、生成幻觉等问题。

需要掌握的技术：

- RAG 参数调优
- 检索召回
- 相关性评估
- 幻觉控制
- 知识库评测

## 阶段完成标准

- 能实现文档问答助手。
- 能解释 RAG 的完整链路。
- 能理解 chunk size、topK 对结果的影响。
- 能让回答附带引用来源。
- 能判断 RAG 问题出在检索侧还是生成侧。

1. 学习 Graph 编排思想。
2. 学习节点、边、状态、条件分支。
3. 学习如何把 Agent 流程拆成多个节点。
4. 学习节点输入输出设计。

6. 学习错误节点和兜底节点。
7. 学习每个节点的日志和观测。

需要掌握的技术：

需要掌握的技术：

- Graph 工作流设计

# 学习原则

1. 不再花大量时间补 Java 基础，直接围绕 Spring AI Alibaba 做 AI 应用闭环。
2. 先跑通 ChatClient，再逐步叠加 Prompt、Memory、Tool Calling、RAG、Graph、MCP。
3. 每个阶段都要有 Demo，不只看文档。
4. Prompt、Memory、Tool Calling、RAG 不要一开始全部混在一起，应该逐步组合。
5. Demo 阶段重功能闭环，生产阶段重安全、观测、评测、成本和权限。
6. Agent 架构设计的核心不是让模型自由发挥，而是让模型在可控流程中完成合适的任务。
7. 企业级 Agent 必须重点关注权限、审计、数据安全、人工确认和回归评测。
